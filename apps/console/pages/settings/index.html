设置 · AI Agent 自动授权中心

配置 Agent 在 Runtime Key 权限内的自动行为与额度。所有资金动作仍受用户规则与链上/账本约束；此处不包含任何私有风控或评分细节。

授权闸门（落地顺序）：先配置资金额度与权限范围并确认责任边界 → 保存服务端策略 → 钱包签名铸造 Runtime Key（不得超过已保存策略）→ Console 完成 Voucher / Settlement → 核对自动化就绪后再导出 handoff 交给 OpenClaw。Voucher 创建/接受不可由 AI 代签。
重要：Runtime Key 不是钱包私钥。它只是 Agent 工作通行证，不能提现、不能转走 USDC、不能修改锁仓额度。请在官方 Console 完成钱包授权；Agent 进程只应持有 Runtime Key。
