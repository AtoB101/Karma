Deploy locally

Run the protocol, static Console shell, and optional testnet scripts on your machine. Production secrets never belong in git — copy docker/.env.example locally after reading SECURITY.md.

Commands (target experience)

git clone https://github.com/AtoB101/Karma.git
cd Karma
cp docker/.env.example .env.local
docker compose -f docker/docker-compose.example.yml up -d
# When published:
# npx karma-agent init
# npx karma-agent connect --network testnet

CLI packages ship from packages/cli when released. API keys are created server-side; do not paste production keys into HTML or front-end bundles.

API Key setup

Issue scoped keys from your operator backend. Public contract calls still require wallet signatures on-chain; HTTP APIs should additionally verify request signatures (openapi/karma-public-console-api.yaml).

Agent ID registration

Use POST /api/public/agents/register (see OpenAPI). Agent identifiers must remain globally unique in your deployment namespace.

Connect to KARMA Trust Layer

On-chain trust layer contracts live under contracts/core/ (canonical: NonCustodialAgentPayment.sol, SettlementEngine.sol). Off-chain receipts and evidence mapping: trusted_agent_runtime/.
