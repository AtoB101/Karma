# Karma Trust Protocol — 公开共建者招募

> **版本**：2026.05 公开版
> **阶段**：Pre-Token Testnet 早期共建期
> **本文不构成任何投资、雇佣、代币承诺或法律要约**。所有权利义务以最终书面协议、适用法律及正式治理程序为准。

Karma 正在构建面向 AI Agent 的可验证执行、证据链与非托管结算的公共信任基础设施。我们诚邀开发者与生态伙伴一起把协议从「可用」推向「可规模化、可信赖」。

---

## 一、招募目标

| 类别 | 人数目标 | 重点方向 | 参与强度 |
|------|----------|----------|----------|
| 核心开发者 | 约 10 名 | 合约、安全、后端、SDK、Console、测试工具链 | 高强度（建议每周 10-20 小时以上） |
| 生态贡献者 / 布道者 | 约 20 名 | 内容、教程、本地化、活动、集成故事、伙伴引荐 | 中强度（每月 3-5 项产出） |
| 公开测试者 | 不设上限 | Testnet 实操、Bug 报告、攻击模拟、工具贡献 | 灵活（待官网与测试入口上线后单独发布详细指引） |

**公开测试者招募将等官网与测试网报名入口完成后单独发布**，优秀测试者（尤其是发现重要漏洞并协助修复者）可直接晋升核心贡献者名单。

---

## 二、核心开发者（约 10 名）

我们寻找能**落地代码、推动测试网闭环**的实干者。以下方向可一人多能，也可多人共担：

1. 智能合约工程师（结算、容量、voucher、升级）
2. 协议安全研究员（威胁建模、审计、Certora/形式化验证）
3. 后端 / API / 结算工程师（状态机、BFF、Webhook）
4. 密码学与身份工程师（EIP-712 收据、委托、proofHash）
5. 可观测性与 SRE（日志、监控、发布流水线）
6. 前端 / Console 工程师（操作端、Wallet Connect、对齐 OpenAPI）
7. SDK 与 Agent 集成工程师（OpenManus / OpenClaw / LangGraph 等）
8. 测试与工具链工程师（E2E、压测、自动化脚本）
9. 技术写作与开发者体验工程师（文档、Sample、15 分钟跑通）
10. 其他高匹配方向（欢迎自荐）

**入选方式**：通过 GitHub PR + 可验证贡献 → 进入核心贡献者档案 → 双方确认后签署贡献协议。

---

## 三、生态贡献者 / 布道者（约 20 名）

侧重**把 Karma 的技术与价值观传递出去**：

- 深度技术文章、视频、直播、教程（优先测试网实操）
- 多语言本地化文档与社区沟通
- 小型 Meetup、黑客松、开源社区活动
- 集成案例与伙伴对接故事
- 高校、钱包、云平台、ISV 等生态联结

**入选依据**：公开可追溯产出（文章链接、活动记录、被合并的文档等）。

---

## 四、贡献值与晋升机制（重要）

我们建立**透明贡献评分体系**（详见后续 `CONTRIBUTION_SCORING.md`）：

- 常规测试 / Bug 报告：5–80 分
- 重要漏洞发现 + 协助修复：200–500+ 分
- 高价值 PR / 工具 / 内容：100–400 分

≥800 分 + 持续贡献 4 周以上 的测试者/贡献者，可申请进入**早期核心贡献者名单**，作为后期权益协商的重要参考。

---

## 五、激励原则（无任何承诺）

1. **当前阶段**：无代币发行计划。
2. **早期贡献者**：对持续关键贡献者，在法律主体与合规就绪后，提供**股权或法律允许的等价长期权益**协商空间（需书面协议 + 尽职调查）。
3. **最终分配**：由透明治理机制（社区 + 利益相关方）共同决定。
4. **测试网激励**：测试资金、任务优先权、公开致谢、贡献档案记录。

所有激励均以正式协议为准，本文档不构成保证。
