# Karma Trust Protocol — 早期共建者招募（公开说明）

> **本文档为公开招募与机制说明，不构成投资、证券或劳动关系的要约。** 具体权利义务以届时签署的协议、适用法律及正式治理程序为准。

Karma 正在构建面向 **AI Agent 的可验证执行、证据链与非托管结算** 的公共信任基础设施。早期阶段我们需要三类关键力量：**安全研究员**、**开发者**与 **推广者 / 生态布道者**，与核心团队一起把协议、实现与生态叙事打磨到可规模化。

---

## 一、我们需要什么样的人

| 方向 | 你在做什么 | 典型产出 |
|------|------------|----------|
| **安全研究员** | 威胁建模、协议与实现审计、形式化与性质验证、测试网与主网前红队 | 审计报告、PoC、Certora/Foundry 性质测试、漏洞负责任披露 |
| **开发者** | 合约、API、运行时、控制台、SDK、测试与工具链 | PR、设计文档、可复现的测试网脚本、开发者体验改进 |
| **推广者 / 布道者** | 把技术与价值观翻译成社区语言：文章、演讲、活动、本地化、伙伴对接 | 教程、案例研究、Meetup、多语种内容、集成故事 |

三者缺一不可：**安全**决定能否承载真实资金与声誉；**开发**决定能否交付；**布道者**决定能否形成可持续的生态与采用。

---

## 二、核心开发邀请（10 人）

以下岗位对应 **约 10 名核心开发贡献者**（可一人多能，也可多人共担同一方向；以贡献协议与代码合并为准，而非职级头衔）。

| # | 岗位方向 | 主要职责（与仓库的对应关系） |
|---|----------|------------------------------|
| 1 | **智能合约工程师** | `karma-core`：结算合约、升级与安全边界、Foundry 测试与 gas 策略 |
| 2 | **协议安全研究员** | 威胁建模、不变量、经济攻击面、审计对接与修复验证 |
| 3 | **形式化验证 / 性质工程师** | 关键路径性质（如容量账本、状态机）与 CI 门禁 |
| 4 | **后端 / 结算与 API 工程师** | `api/`、`core/settlement/`：状态机、容量、收据与运行时安全 |
| 5 | **密码学与身份工程师** | 收据签名、EIP-712、密钥与委托、与链上 `scopeHash` / `proofHash` 对齐 |
| 6 | **可观测性与 SRE** | 日志、指标、追踪、测试网演练、发布与回滚 |
| 7 | **前端 / 控制台工程师** | `apps/console/`：操作端、状态同步、对 openapi 的对齐 |
| 8 | **SDK 与 Agent 适配工程师** | `packages/*`、`examples/`：OpenManus/OpenClaw 等适配与示例 |
| 9 | **测试与工具链工程师** | 端到端测试、测试网脚本、可重复的一键流程与文档 |
| 10 | **技术写作与开发者体验** | OpenAPI、Runbook、多语言文档、Sample 与「15 分钟跑通」路径 |

**参与方式（原则）：**  
以 **GitHub 公开协作** 为主：Issue / Discussion 对齐方向 → 小步 PR → Code Review → 合并。核心席位不依赖地域，依赖 **可验证贡献与沟通纪律**。

## 三、生态布道者 / 宣传者（20 名）

**约 20 名生态布道者** 侧重 **传播、教育与联结**（不替代安全与工程门禁）。

包括但不限于：

- **技术内容**：深度文章、视频、直播、系列教程（测试网实操优先）。
- **本地化**：英文以外的文档与社区沟通（质量优先）。
- **活动与社群**：小型 Meetup、黑客松赛道、高校与开源社区联络。
- **集成叙事**：把「Agent + 证据 + 结算」讲给钱包、云平台、ISV 与合规团队。
- **伙伴线索**：引荐潜在集成方（最终合作由项目方与法务流程承接）。

布道者同样以 **可追溯产出**（公开链接、活动记录、被采纳的文档修订）作为参与证明，便于后期纳入贡献档案。

## 四、激励与长期机制（原则性说明）

1. **代币经济模型**  
   项目 **在后期** 将结合合规与产品阶段，评估是否引入 **代币** 类激励；**当前未承诺任何代币发行、销售或空投时间表**。若未来启动，应以正式公告与法律文件为准。

2. **早期参与与股权（或等价权益）**  
   对 **持续、关键且经双方确认的早期贡献者**，项目方计划在法律与主体结构就绪的前提下，提供 **股权或法律结构允许的等价长期权益** 的协商空间。  
   **重要：** 是否授予、授予范围与条件取决于 **尽职调查、管辖法律、公司章程序列与书面协议**；本文不构成任何要约或保证。

3. **最终分配由整个生态决定**  
   长期而言，重大激励与资源分配（包括但不限于贡献认可、预算与公共品资助）应逐步过渡到 **开放治理**：由贡献者社区与利益相关方在透明规则下共同决定。具体治理形态（基金会、DAO、委托投票或混合结构）将在路线与合规评估后单独发布，而非由单篇文章定义。

## 五、如何联系与下一步

- **开发者 / 安全**：建议从仓库 **Issues / Discussions** 自我介绍并附上过往公开作品（Repo、报告、演讲材料）。  
- **布道者**：可附 **3 条代表作链接** + 希望覆盖的语言/地区 + 每月可投入时间量级。

我们会在能力范围内回复；若短期内未回复，通常表示当前席位与方向暂不匹配，**不代表对你能力的否定**。

---

## 六、价值观与安全

- **默认公开、可审计**：能公开的代码与文档尽量公开；敏感规则与风控策略在 **私有仓与合规边界** 内处理。  
- **安全优先于叙事**：未经验证的上链与资金路径不会为宣传而提前承诺。  
- **尊重用户与贡献者数据**：最小化收集，明确用途，遵守适用隐私法律。
